- **Análisis de Opciones:** 
  1. *LiteLLM `drop_params: true`:* Bloqueo a nivel de proxy (reactivo).
  2. *Parche JavaScript en OpenClaw:* Script de inyección `patch_openclaw_memory.js` para purgar metadatos (frágil y genera deuda técnica).

- **Acción Tomada:** Se configuró el Gateway de OpenClaw separando los bloques. 
  - Se eliminaron los cruces `openai/vertex_ai/*` en la UI.
  - Se etiquetaron explícitamente los de OpenAI como "(AI Studio)" y se creó un bloque nativo `vertex_ai` de primer nivel que apunta al proxy.
